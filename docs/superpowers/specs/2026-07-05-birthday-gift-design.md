# 杨丽生日互动网页设计规格

## 目标

制作一个手机优先的单页生日礼物，通过 GitHub Pages 链接打开。页面复刻用户确认的粉色生日蛋糕视觉，并通过麦克风吹气或点击蜡烛触发祝福揭晓。

## 视觉状态

### 点燃状态

- 视口以 `390 × 844` 为设计基准，页面铺满 `100dvh`。
- 顶部显示手写风格的 `To 杨丽`，`To` 较小，`杨丽` 较大，配玫红下划线与两处黄色光点。
- 中部显示完整白色蛋糕和一支点燃的粉色蜡烛。
- 底部显示 `吹灭蜡烛，许个愿吧`、麦克风图标和音量波形。
- 右上角提供低调的声音开关。

### 揭晓状态

- 移除 `To 杨丽`、麦克风提示和波形。
- 蜡烛熄灭，保留一缕烟。
- 中文祝福作为主标题：`又长大一岁，但你仍然是时间流逝中的例外。`
- 英文作为副标题：`ANOTHER YEAR. STILL THE EXCEPTION.`
- 不使用弹窗卡片；通过两张同构场景图交叉淡入淡出完成整屏揭晓。

## 交互与声音

- 用户点击底部提示后请求麦克风权限并开始检测吹气。
- 连续高于阈值约 180ms 时视为吹气成功，切换至揭晓状态。
- 点击蜡烛始终可作为备用操作；麦克风拒绝或不可用时显示简短提示。
- 不播放持续背景音乐。揭晓时播放约 1 秒的柔和三音提示音。
- 声音默认开启，但提供可访问的声音开关。

## 技术约束

- 使用 Vite、TypeScript 和原生 DOM/CSS，不引入 UI 框架。
- 两张确认后的手机视觉作为完整场景资源，保持设计一致性。
- 使用 `100dvh`、安全区 inset 和 `object-fit: cover` 适配常见手机屏幕。
- 麦克风仅在 HTTPS 或 localhost 下启用；GitHub Pages 满足 HTTPS 要求。
- 页面无后台、无表单、无分析追踪，不上传或保存任何麦克风音频。

## 验收标准

- 390×844、393×852、430×932 三种手机视口下无文字或蛋糕裁切。
- 麦克风允许时可通过吹气揭晓；拒绝时可点击蜡烛揭晓。
- 揭晓只发生一次，切换平滑，提示音不会循环。
- iOS Safari 与 Android Chrome 可打开并完成核心交互。
- GitHub Pages 链接可公开访问。
